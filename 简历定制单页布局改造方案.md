# 简历定制页面 — 单页左右分栏布局改造方案

## 现状分析

当前 `/customize` 页面使用**三步向导**模式：

```
Step 1: 选择简历  →  Step 2: 填写岗位信息  →  Step 3: 确认并生成  →  查看结果
```

### 存在的问题

1. **步骤割裂** — 用户需要点 3 次"下一步"才能完成操作
2. **无法实时预览** — 必须等第三步点击生成后才能看到结果
3. **修改不便捷** — 想改岗位信息需要"返回上一步"，然后重新生成
4. **信息不直观** — 第三步确认页面只是单纯罗列已填信息，浪费空间

### 当前布局结构

```
┌─ AppLayout ──────────────────────────────────────────────────┐
│ ┌──────────┬───────────────────────────────────────────────┐ │
│ │          │ header: "简历定制"                             │ │
│ │ sidebar  ├───────────────────────────────────────────────┤ │
│ │ 240px    │ max-w-5xl 居中容器                             │ │
│ │          │  · 步骤指示器 (①②③)                            │ │
│ │          │  · 当前步骤内容区                               │ │
│ │          │  · 上一步 / 下一步 按钮                         │ │
│ │          │                                               │ │
│ │          │ GenerateResult (全宽, 仅步骤3生成后可见)         │ │
│ │          │  · 左: Markdown 编辑  · 右: 预览               │ │
│ └──────────┴───────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
```

---

## 改造方案

### 设计目标

1. **一个页面搞定** — 去掉步骤切换，所有输入区始终可见
2. **预览占 ≥50% 宽度** — 生成的简历在右侧大面板中预览，所见即所得
3. **保留全部功能** — 保存、投递、导出 MD/TXT、Markdown 源码编辑

### 新布局

```
┌─ AppLayout ──────────────────────────────────────────────────┐
│ ┌──────────┬────────────────────────────────────────────────┐ │
│ │          │ header: "简历定制" + 副标题                      │ │
│ │ sidebar  ├─────────────────────┬──────────────────────────┤ │
│ │ 240px    │  左侧面板 ~42%       │  右侧面板 ~58%            │ │
│ │          │  (overflow-auto)    │  (flex-1, 独立滚动)      │ │
│ │          │                     │                          │ │
│ │          │ ⚠️ API Key 警告     │  [ 预览 │ Markdown 源码 ] │ │
│ │          │  (可关闭)           │                          │ │
│ │          │                     │  空态:                   │ │
│ │          │ 📄 选择已有简历      │  填写左侧信息并点击生成    │ │
│ │          │ [下拉框 ▼]         │  即可预览定制简历          │ │
│ │          │ 简历内容预览        │                          │ │
│ │          │                     │  生成后:                  │ │
│ │          │ 💼 岗位信息          │  渲染的 Markdown 简历     │ │
│ │          │ ┌──────┬──────┐    │  (prose-resume 样式)     │ │
│ │          │ │公司*  │职位*  │    │                          │ │
│ │          │ ├──────┴──────┤    │                          │ │
│ │          │ │ 岗位描述*    │    │                          │ │
│ │          │ ├──────────────┤    │                          │ │
│ │          │ │ 公司信息     │    │                          │ │
│ │          │ └──────────────┘    │                          │ │
│ │          │                     │                          │ │
│ │          │ [🪄 一键生成定向简历]│                          │ │
│ │          │                     │                          │ │
│ │          │ 生成后操作:          │                          │ │
│ │          │ [💾 保存] [📤 投递] │                          │ │
│ │          │ [📥 导出MD] [📥 TXT]│                          │ │
│ └──────────┴─────────────────────┴──────────────────────────┘

移动端 (<1024px): 左右面板上下堆叠
```

### 各区域详细说明

#### 左侧面板 (w-[42%], 独立滚动)

| 序号 | 区域 | 说明 | 始终可见 |
|------|------|------|:---:|
| 1 | API Key 警告 | 黄色卡片 + 前往设置按钮 + X 关闭 | 条件 |
| 2 | 选择已有简历 | 下拉框 (含来源标签) + 前500字预览 | ✅ |
| 3 | 岗位信息表单 | 公司名称* + 职位名称* (双列) / 岗位描述* / 公司信息 | ✅ |
| 4 | 生成按钮 | accent 绿色按钮，条件启用，加载时显示动画 | ✅ |
| 5 | 操作按钮组 | 保存 / 添加到投递 / 导出MD / 导出TXT | 生成后 |

#### 右侧面板 (flex-1, 独立滚动)

| 状态 | 展示内容 |
|------|---------|
| **空态** | 大图标 + "填写左侧信息并点击生成即可预览定制简历" |
| **加载中** | 旋转动画 + "AI 正在为你定制简历..." |
| **错误** | 红色卡片 + 错误信息 + 重试按钮 |
| **成功** | Tab 切换：「预览」/「Markdown 源码」 |

#### Tab 栏细节

- **预览 tab**: 使用现有 `prose-resume` 样式渲染 Markdown，独立滚动
- **源码 tab**: textarea 双向绑定，编辑实时更新预览
- 右侧显示字符计数

## 技术实现

### 改动范围

| 操作 | 文件 | 说明 |
|------|------|------|
| **重写** | `src/views/Customize.vue` | 合并 GenerateResult 全部功能，改为左右分栏 |
| **删除** | `src/components/Customize/GenerateResult.vue` | 功能已内聚到 Customize.vue |

### 不改动的文件

`stores/ai.ts`、`stores/resume.ts`、`utils/markdown.ts`、`utils/ai.ts`、`utils/message.ts`、`router/index.ts`、`style.css`、`types/index.ts` — 全部保持不变。

### Script 层改动

**删除:**
- `currentStep` ref、`steps` 常量
- `canNext()`、`nextStep()`、`prevStep()` 函数
- `GenerateResult` 组件导入
- `ChevronLeft`、`ChevronRight` 图标

**新增:**
- `activeTab` ref (`'preview' | 'source'`) — 右侧面板 Tab 切换
- `showApiKeyWarning` ref (`true`) — 警告可关闭
- `renderedPreview` computed — 调用 `renderMarkdown()`
- `canGenerate` computed — 生成按钮启用条件
- `exportMd()` / `exportTxt()` — 从 GenerateResult 移入
- `dismissApiKeyWarning()` — 关闭警告
- 图标: `Eye`, `Save`, `FileDown`, `Send`, `X`

**保留不变:**
- 所有表单字段 refs
- `getSourceTypeLabel()`、`handleGenerate()`、`saveAsNewResume()`、`goToApplication()`、`goToSettings()`
- `onMounted` 中 `loadResumes()` + `?resumeId=xxx` 快速入口
- Markdown 代码块清理逻辑

### 关键边界处理

| 场景 | 处理方式 |
|------|---------|
| API Key 未配置 | 顶部黄色警告（可X关闭），生成按钮禁用 |
| 无简历 | 下拉框提示文字，生成按钮禁用 |
| 生成按钮启用条件 | 已选简历 + 公司名称 + 职位名称 + 岗位描述 均非空 + hasApiKey |
| 快速入口 `?resumeId=xxx` | 下拉框预选对应简历，用户仍需手动填写岗位信息并点击生成 |
| 重新生成 | 清空旧内容 → 右侧加载态 → 新内容替换 → 自动切回预览 Tab |
| 保存后 | `savedSuccess` flag 解锁「添加到投递管理」按钮 |
| Markdown 代码块清理 | 保存/投递时剥离 \`\`\`markdown 包裹 |

## 与现有方案的对比

| 维度 | 当前（3步向导） | 改造后（左右分栏） |
|------|:---:|:---:|
| 操作步骤 | 3步 + 确认 | 1页完成 |
| 预览可见性 | 第3步生成后才见 | 始终可见（右侧面板） |
| 修改岗位信息 | 返回上一步 → 下一步 → 重新生成 | 直接修改左侧 → 重新生成 |
| 预览宽度占比 | 第3步: 约50%（编辑+预览） | **≥55%** 纯预览区 |
| 信息密度 | 低（每步一屏） | 高（所有信息一站式） |
| 步骤指示器 | ✅ 有 | ❌ 去掉 |
| Markdown 编辑 | ✅ | ✅（源码 Tab） |
| 保存/导出/投递 | ✅ | ✅ 完全相同 |

## 预期效果

1. 用户打开页面即看到所有输入区和空白预览区
2. 从简历管理页点击"定制简历"，跳转后自动预选简历
3. 填写岗位信息后点击"一键生成"，右侧立即显示预览
4. 不满意可修改岗位信息后点击"重新生成"
5. 可切换到"Markdown 源码"Tab 微调内容
6. 保存后可直接跳转到投递管理
